Here's a structured plan that builds on the roadmap from before, but now with specific time estimates, milestones, and a logical weekly flow.

The core principle: **never learn a skill in isolation**. Every week ends with something you actually built or shipped.Expand each phase to see the week-by-week breakdown. Switch the pace selector to see how the timeline changes for your schedule.

A few things that will make or break this plan:

**Every week ends with a project, not a quiz.** Abstract concepts don't stick until you've broken something and fixed it. The projects are designed so each one uses the skills from that week in a way that would actually be useful to you.

**Git from week 2 onwards is non-negotiable.** Every project you build after that gets committed. By the end you'll have a GitHub profile with 10+ repos showing real progression — that's more valuable than any certificate.

**Python and JavaScript are mirrors of each other.** When you hit Phase 4, you'll notice that `async/await`, error handling, and data validation all work the same way conceptually. You're not learning them twice — you're seeing the same ideas in a second language, which is much faster.

**The capstone project in Phase 6 is the whole point.** It combines everything: Python backend, TypeScript frontend, Docker, Git history, a proper README. That single project demonstrates more than a month of isolated exercises would.

Realistic expectation at 10 hrs/week: you'll feel genuinely productive around week 6 (end of Python), and you'll feel like a developer around week 14. Click any skill card in the weeks above to dive straight into that topic.
